# 🔐 비밀번호 보호 설정 가이드

이 가이드를 따라하시면 사이트에 비밀번호 잠금이 걸려요.
**환경변수**라는 안전한 방식으로 비밀번호를 저장합니다.

---

## 📌 작동 방식

```
사이트 접속
    ↓
"비밀번호 입력" 화면 등장
    ↓
올바른 비밀번호 입력
    ↓
24시간 동안 자동 유지
    ↓
24시간 후 다시 로그인 필요
```

---

## 🎯 비밀번호 정하기

먼저 사용할 비밀번호를 정하세요. 추천:
- ✅ 8자 이상
- ✅ 본인이 기억하기 쉬운 것
- ❌ "1234", "password" 같은 너무 단순한 것 X

예시: `MyStock2026!`, `Stock@MJ123` 등

---

## 1단계: Render에 비밀번호 설정

Render 사이트에서 환경변수를 설정해요. 코드에는 비밀번호가 들어가지 않아서 안전합니다.

### 1-1. Render 대시보드 접속
1. https://dashboard.render.com 접속
2. 본인 서비스 (`stock` 또는 `pullback-scanner`) 클릭

### 1-2. Environment 탭으로 이동
1. 왼쪽 메뉴에서 **"Environment"** 클릭
2. **"Add Environment Variable"** 또는 **"+ Add"** 버튼 클릭

### 1-3. 변수 추가
다음과 같이 입력:

| 항목 | 값 |
|------|-----|
| **Key** | `APP_PASSWORD` |
| **Value** | (정하신 비밀번호 입력) |

⚠️ Key는 정확히 `APP_PASSWORD` 여야 해요!

### 1-4. 저장
1. **"Save Changes"** 클릭
2. Render가 자동으로 재배포 시작 (3~5분)
3. 재배포 완료 후 사이트 접속 시 비밀번호 화면 등장!

---

## 2단계: 로컬 PC에 비밀번호 설정 (선택)

PC에서도 보안 적용하고 싶으면 설정. 본인만 쓰시면 안 해도 OK.

### 방법 A: 환경변수로 설정

1. 윈도우 검색 → "환경 변수" 검색
2. **"시스템 환경 변수 편집"** 클릭
3. **"환경 변수"** 버튼 클릭
4. 사용자 변수 영역에서 **"새로 만들기"** 클릭
5. 입력:
   - 변수 이름: `APP_PASSWORD`
   - 변수 값: (Render와 같은 비밀번호)
6. **확인**
7. PC 재부팅 (또는 cmd 새로 열기)
8. `run.bat` 실행

### 방법 B: run.bat 수정 (간단)

`run.bat` 파일을 메모장으로 열고, `python app.py` 줄 위에 한 줄 추가:

```batch
set APP_PASSWORD=YourPasswordHere
python app.py
```

⚠️ 이렇게 하면 `run.bat` 파일에 비밀번호가 적혀요. 본인만 쓰는 PC면 OK, GitHub 업로드 시 주의!

### 방법 C: 로컬은 비밀번호 안 걸기 (가장 편함)

PC에서는 `APP_PASSWORD` 환경변수를 안 만들면, 자동으로 인증 비활성화 모드로 작동해요. 본인 PC라 보안 필요 없으면 이 방법이 가장 편해요.

---

## 3단계: 첫 로그인 테스트

1. 사이트 접속 (`stock-xjuu.onrender.com`)
2. 비밀번호 입력 화면 등장 ✨
3. 정한 비밀번호 입력 → "접속하기"
4. 정상 로그인 시 사이트 사용 가능
5. 우측 상단에 **"로그아웃"** 버튼 등장

## 🤔 자주 묻는 질문

### Q. 비밀번호 잊어버리면?
- Render Environment 탭에서 `APP_PASSWORD` 값 확인 가능
- 또는 새 비밀번호로 변경 (Edit → 저장)

### Q. 비밀번호 바꾸고 싶으면?
1. Render → Environment → `APP_PASSWORD` 클릭
2. 값 수정 → 저장
3. 자동 재배포
4. 새 비밀번호로 다시 로그인

### Q. 24시간이 너무 짧아요
`app.py`에서 `TOKEN_LIFETIME_HOURS = 24` 부분을 원하는 시간으로 변경:
- `1`: 1시간
- `24`: 하루
- `168`: 1주일
- `720`: 한 달

### Q. PC와 핸드폰이 다르게 작동해요?
정상이에요. 각 기기/브라우저별로 별도 토큰 발급. 각각 한 번씩 로그인 필요.

### Q. 로그아웃 안 하고 그냥 닫아도 되나요?
네, 24시간 후 자동 만료돼요.

### Q. 누가 비밀번호 추측해서 들어오면?
- 비밀번호 시도마다 0.5초 딜레이 (브루트포스 방지)
- 1시간에 7,200번 시도 가능 (이론적으로)
- 8자 이상 비밀번호면 사실상 불가능

## 🚨 문제 해결

### 비밀번호 화면이 안 나와요
1. `app.py`가 v6 (인증 추가된 버전)인지 확인
2. Render Environment에 `APP_PASSWORD` 키가 정확한지 확인
3. 재배포가 완료됐는지 확인 (Logs 탭)

### "비밀번호가 올바르지 않습니다"
- 대소문자 확인
- 공백 확인 (앞뒤 공백 주의)
- Render Environment의 값과 정확히 일치해야 함

### 로그인했는데 401 에러
- 브라우저 쿠키가 차단되어 있을 수 있음
- 시크릿 모드면 매번 로그인 필요
- 다른 브라우저로 시도
